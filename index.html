HTML

  • The head element contains information about the webpage.
  • There are six head elements, "h1" is the largest and 6 is the smallest, you should only use "h1" once
  • The body element represents the visible content shown to the user.
  • The "p" element represents a paragraph or block of text
  • The "br" element creates a line break between two blocks of content
  • The "a" element is an anchor and creates links to the same or other web pages
  • "href" is an attribute which will point to a URL for a link, you need to consider this for things like CSS style sheets
  • There are three different list types; "ul" is an unordered list, "ol" is an ordered list and "li" allows you to list items
  • The "img" element contains information about images displayed on the webpage and the "src" attribute is most important here as it defines the location of the image

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A box-shadow adds shadows around an elements frame
  • A selector defines the element or attributes to which the rules, or declarations, will apply
  • Declarations contain two important components: the CSS property we want to apply and the value of the property e.g. "color:pink" - where "color" is the CSS property and "pink" is the value
  • You can use the "*" wildcard symbol to apply a rule to all elelments visible on the page
  • You can combine two selectors into a single CSS rule with a comma, allowing the declarations to affect both selectors
  • The above helps us write DRY (Do not Repeat Yourself) Code
  • A text-align is a way of positioning a title or heading on a web page
  • "The "display" property can be assigned with a block value to assign images on new lines

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout main switches to the main branch
  • git pull origin main pulls in the latest changes

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • We can use the "console.log" method to output a message to the web console by adding an argument
  • The script file is linked with the "body" element
  • Control flow is the order in which a computer executes code in a script.
  • The order of the code affects the outcome, Javascript is read fro,m top to bottom, and order counts
  • We must be intentional about how we set up code
  • Conditional statements and loops can be used to control the order in which code is executed
  • When a computer encounters and if statement, it will check to see if the condition is true or truthy, the computer will ignore whatever is in between the curly brackts if the value isnt true or has a value that isn't truthy
  • The strict equality operator "===" checks to see if two values are equal and returns a boolean result (true for equal, false for not)
  • We can use an array to change the value of our variable in larger sets of data as opposed to manually adjusting them
  • An index identifies the location of the data item in an array
  • Arrays are zero indexed and sequential
  • Loops are efficient ways to change the values of the index
  • A "for" loop can be used to log each topic from an array in the console
  • It uses the predictable pattern of indices to perform a task on all items in an array by allowing a single code block to be executed over and over
  • The first statement determines the starting point for a loop
  • The second stateent is a condition, as long as the number in the variable "x" is LESS than the length of the array, the loop will keep running
  • The final statement is what allowas the array to iterate over each item, the code "x++" is a shortcut for "x+1"
  • Don't forget the curly brackets!
